# RNN
## Concepts
Need to deal with sequential data. Traditional machine learning methods fails.
The time element of RNN can act as a feature. Different data can be processed by one neuron through . 

**Sequence Modeling**
- Many to One (Input is Sequence) -> Sentiment analysis
- One to Many (Output is sequence) -> Image captioning
- Many to Many

**Dataflow of RNN**
Forward and Backward link. One neuron takes input from the neurons that preceeds it for the current time step, and also input from itself from the previous timestep.
Every recurrent layers except for the very last **must return** sequence as output. The last layer is flexible.

One timestep repreresents one item of the sequential data. And for this reason, you can unroll the timesteps so that each one takes a specific input form the sequential data.
![[Pasted image 20260423143744.png|400]] 
Note that each unfolded item share their parameters.
$$\text{Params} = (n \times m) + (m \times m) + m$$

**Caveat of Basic RNN**
Vanishing/Exploding Gradient for long sequence of inputs.
To understand, at each timestep, we will multiply the first input by the feedback weight. This causes exponential explosion if weight > 1 and vanishing gradient if weight < 1. 
![[Pasted image 20260423153521.png|200]]

For example, if $W_{2}=2$, 2ith 50 item of sequential input, $\text{input}_{1}$ will be multiplied by $2^{50}$. This will affect the gradient update step. Similar problems occur for $W_{2}<1$ (vanishing gradient).
**Solution:** Long Short-Term Memory (LSTM).

**Dimensions**
$W_{xh} \rightarrow$ hidden neuron $\times$ input dimension
$W_{hy} \rightarrow$ output neuron $\times$ hidden neuron
# Long Short-Term Memory
In vanilla RNN, we only use short-term memory, which causes gradient problems. LSTM solves this by using a long-term memory architecture.
![[Pasted image 20260423154857.png|300]]
[Illustration of LSTM]

Uses Sigmoid (0, 1) and Tanh (-1, 1) activation at different parts.

**Components of LSTM**
- Cell State (C) Long Term - Isn't affected by any weights
- Hidden State (h) Short Term
- Each neuron/cell is made up of:
	- Forget Gate (Sigmoid) - how much long term memory to remember
	- Candidate Gate (Sigmoid x Tanh) - update the long term memory
		- Input Gate: The sigmoid part
		- Candidate State: This is the tanh paired with the input gate
	- Output Gate (Sigmoid -> Tanh) - update short term, doesn't affect long term memory

>[!note]
> All sigmoids are called gates, while tanh are called memory or state (observation/conjecture).
>
Other gates affects the short term memory through updating the long term memory (Indirect).


The role of Sigmoid is to regulate the percentage of long term information that is allowed to pass/keep (remember).
Role of Tanh is to actually process the information.

**Calculations**
Setiap gate memiliki bobot sendiri, bobot $U$ adalah untuk input, bobot $W$ akan dikalikan dengan hidden state.

Every calculations marked with the pink circles are done element wise (hadamard).
Linear combination before activation uses dot products.

>[!important]
>The output of LSTM is the hidden state, which every sequence will have their own output, and you can take all of them (many to many) or only the final (many to one).

**Learnable Parameters**
Parameters per gate:
$$(h\times d) + (h \times h) + h$$
Where $h = \text{hidden state dimension}$
and $d = \text{input dimension}$

Which means total parameters for the gates are $PPG \times 4$

Parameters in the output layer:
$$(h\times y)+y$$
