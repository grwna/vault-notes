# Why CNN?
FFNN uses matrix multiplication which only works for flat data. This is disadvantageous for images, since images (especially colored ones) have multiple layers of data (e.g RGB).

CNN uses convolution which allows for multiple levels of data at any layers of neural network.

# Local Connectivity & Parameter Sharing
Both these concepts serves to limit how the neural network operates. It makes the computation exponentially more efficient, while also improving generalization.
## Local Connectivity
As opposed to being fully-connected (all neurons in one layer is connected to all neurons in previous/next layers). Local connectivity only connects neurons to some of the neurons in the previous layers (called *receptive fields*). This concept more closely resembled the brain of an organism.

**Kernels**
A kernel is a small matrix that acts as a sliding window (filter) that convolves over the whole dataset. This window acts as the receptive field connector, only the neurons inside this window are connected. 
## Parameter Sharing
Without parameter sharing, every spatial location uses its own filter (kernel). With it, they might only use the same one kernel.

**Traditional:** if there are $m$ inputs and $n$ outputs, then there are $(m+1)\times n$ parameters, including bias.
**Parameter sharing**: if there are $m$ inputs and $n$ outputs, and kernel width is $k$ then there are $k\times 1$ parameters.

# Convolutional Layer
**Complex Layer Terminology**
- Input to layer
- *Convolutional Layer:*
	- Convolution stage
	- Detector stage
	- Pooling stage
- Next layer

**Complex Layer Terminology**
- Input to layer
- Convolution layer
- Detector layer
- Pooling layer
- Next layer

CNN architectures are usually: Input -> Convolutional Layer -> Fully Connected Layer -> Output
with activations and pooling oftentimes inserted in between

## Components in Convolutional Layer
**Convolution Stage**
- Feature Extraction
- Stride (amount of pixels the kernel moves)
- Padding (layer of zeroes around input)
- Output dimension
	The spatial size of output is $V\times V\times K$  (for square image), where: 
	- $K$ - number of kernels (will be equal number of outputs)
	- $V=\frac{W-F+2P}{S}+1$
		- $W$ - width of image
		- $S$ - stride (affect's the output size)
		- $P$ - padding
		- $F$ - width of kernel
**Note:** more conv layer -> more complex extracted feature

>[!note]
>Kernel is equal to the weight matrix

**Detector Stage**
This is the "activation stage" that introduces non-linearity to a system that has only been computing linear operations during convolution stage.
Functions:
- ReLU (primarily)
- Sigmoid
- Tanh

**Pooling Stage**
Reduces spatial size of output -> filters can explore bigger parts of image.
This reduces overfitting. 
Pooling types:
- Max Pooling
- Average Pooling
- L2-Norm Pooling

**Flattening**
Before the data is passed to FC-layer, it has to be flattened first.
